## P0 - Инструменты и автоматизация

- [x] Закрыть verification gap: запускать `tools/CoreSmoke` в автоматизации (CI), а не только статически. GitHub Actions workflow `.github/workflows/core-smoke.yml` ставит pinned .NET 8 SDK (`actions/setup-dotnet@v4`, `8.0.x`) и запускает `scripts/run-core-smoke.sh` на каждый push/PR, затрагивающий `Packages/com.runechess.core/**`, `tools/CoreSmoke/**`, сам workflow или скрипт (+`workflow_dispatch` для ручного запуска). Скрипт `scripts/run-core-smoke.sh` — единый источник правды для локального и CI-прогона: проверяет наличие `dotnet`, собирает и выполняет `CoreSmoke` в Release; `Require`/`RequireThrows` бросают `InvalidOperationException` при провале, поэтому ненулевой exit ломает сборку, а чистый выход = все проверки прошли. Это устраняет повторяющийся «verification gap: .NET SDK недоступен» из десятков завершённых задач. ВЕРИФИКАЦИЯ: .NET 8.0.128 установлен в сессии (apt), `./scripts/run-core-smoke.sh` локально проходит («Core smoke checks passed.», exit 0). Реальный прогон GitHub Actions подтвердится после первого push. Декомпозиция top-P0 из `tasks/.suggestions.md` («Install a pinned .NET SDK or Unity Editor CI image so tools/CoreSmoke compiles in automation»).
